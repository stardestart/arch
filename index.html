Установка Arch Linux (0т новичка, таким же новичкам).

Предисловие.

У меня не было опыта работы с GNU/Linux системами, но появилось желание поменять Windows на что-нибудь другое. Попробовав массу различных дистрибутивов GNU/Linux, я решил учиться Linux с помощью Arch Linux, он понравился мне максимальным минимализмом и схожестью с детским конструктором собирая и разбирая который ты узнаешь для чего нужна каждая деталь. Методом проб и ошибок я определил необходимый мне минимум пакетов для повседневной работы. Скопив относительно большой объем информации, появилась необходимость создания шпаргалки\напоминалки, через некоторое время я решил поделиться своим опытом с такими же новичками в linux, как и я сам, на понятном для новичков языке. Важно знать, что после этой установки ВСЕ ДАННЫЕ с дисков будут УДАЛЕНЫ.

Подготовка к установке.

  1. Заходим на wiki.archlinux если там изложено понятно, то лучше использовать официальную wiki.

  2. Определяем какой режим загрузки использует ваш ПК, UEFI или только Legacy Boot (Для Windows: Win + R, команда «msinfo32» в строке «Режим BIOS»).

  3. Скачиваем образ ArchLinux.

  4. Скачиваем rufus, для создания загрузочной флешки.

  5. Записываем образ на флешку, при перезагрузке загружаемся с флешки (Клавиша F12, F11 или Esc, зависит от производителя оборудования). Если есть uefi, то в меню выбираем загрузку через uefi.

  6. Для установки необходимо постоянное интернет соединение, это одна из особенностей Arch, он устанавливает обновленные пакеты. Если подключение к интернет-сети через провод, то скорее всего интернет будет работать из «коробки», если по wifi то открываем выпадающий список ниже.

    Если ПК подключён к сети через Wi-Fi.

    1. Подключаем устройство wlan0 (скорее всего оно у вас одно) к wifi-сети ИмяСети.

      #: iwctl station wlan0 connect ИмяСети

    2. Вводим пароль wifi-сети.

      #: ПарольСети

    Проверка соединения, с помощью команды ping выполним 3 раза прозвон ya.ru.

    #: ping -c 3 ya.ru

  7. Выбираем русскую раскладку.

    #: loadkeys ru

  8. Установка шрифта с кириллицей.

    #: setfont ter-v18n

  9. Определяем часовой пояс.

    #: timedatectl set-timezone "$(curl https://ipapi.co/timezone)"

Разметка и форматирование носителей.

  1. Просмотрим диски и их разделы. В Windows диски/разделы помечаются буквами A (floppy), B (что-то забытое и ненужное), C (обычно диск/раздел для операционной системы), D и тд., а в linux сам диск получает метку sda, sdb, sdc (nvme0n1, nvme1n1), а разделы диска получают метку по принципу sda1, sda2, sda3; sdb1, sdb2, sdb3; sdc1, sdc2, sdc3 (nvme0n1p1, nvme0n1p2; nvme1n1p1, nvme1n1p2).

    #: fdisk -l

  2. Перейдем в будущий системный диск, пусть это будет sda.

    #: fdisk /dev/sda

  3. Далее выбираем пункт, подходящий для вашего ПК. Современные ПК имеют поддержку UEFI, лучше выбирать именно этот пункт, он подразумевает отказ от дополнительной установки GRAB-а. В случае отсутствия UEFI, выбирайте LEGACY BOOT с GPT (GPT разметка более современная).

    Для legacy boot с MBR структурой диска.

    1. Установим на диске MBR структуру.

      #: o

    2. Создадим первый раздел, GRUB-раздел.

      #: n

    3. Выбираем тип раздела PrimaryPartition.

      #: p

    4. Номер раздела (если «1» по умолчанию, можно сразу нажать ENTER).

      #: 1

    5. Первый сектор (если «2048» по умолчанию, можно сразу нажать ENTER).

      #: 2048

    6. Размер GRUB-раздела.

      #: +512m

    7. Подтверждаем.

      #: y

    8. Установим Boot-флаг загрузки.

      #: a

    9. Создаем второй раздел, SWAP-раздел (Раздел подкачки), я встречал очень много комментариев, что если использование спящего режима не планируется, то этот раздел не нужен, но сам я не берусь это подтверждать или опровергать.

      #: n

    10. Выбираем тип раздела PrimaryPartition.

      #: p

    11. Номер раздела (если «2» по умолчанию, можно сразу нажать ENTER).

      #: 2 
    12. Размер SWAP-раздела устанавливаем по формуле:

    13. Подтверждаем.

      #: y

    14. Создаем третий раздел, ROOT-раздел.

      #: n

    15. Выбираем тип раздела PrimaryPartition.

      #: p

    16. Номер раздела (если «3» по умолчанию, можно сразу нажать ENTER).

      #: 3

    17. Подтверждаем.

      #: y

    18. Записываем изменения на диск.

      #: w

    19. Форматируем GRUB-раздел с меткой «boot».

      #: mkfs.ext2 /dev/sda1 -L boot

  4. Форматируем sda2 (SWAP-раздел) с меткой «swap»; sda3 — для legacy boot с gpt.

    #: mkswap /dev/sda2(3) -L swap

  5. Форматируем sda3 (ROOT-раздел) с меткой «root»; sda4 — для legacy boot с gpt.

    #: mkfs.ext4 /dev/sda3(4) -L root

  6. Перейдем во второй физический диск, если он есть.

    #: fdisk /dev/sdb

  7. Установить на диске GPT или (MBR) структуру, лучше GPT.

    #: g(o)

  8. Записать изменения на диск.

    #: w

  9. Форматируем диск с меткой «data».

    #: mkfs.ext4 /dev/sdb -L data

  10. Монтируем sda3 (ROOT-раздел) к директории /mnt; sda4 — для legacy boot с gpt.

    #: mount /dev/sda3(4) /mnt

  11. Создаём папки «boot, data» в директории /mnt.

    #: mkdir /mnt/{boot,data}

  12. Монтируем sda1 (BOOT-раздел) к директории /mnt/boot.

    #: mount /dev/sda1 /mnt/boot

  13. Монтируем sda2 (SWAP-раздел); sda3 — для legacy boot с gpt.

    #: swapon /dev/sda2(3)

  14. Монтируем второй физический диск к директории /mnt/data.

    #: mount /dev/sdb /mnt/data

  15. Смотрим результат.

    #: lsblk -f

Установка ОС.

  1. Установка системы и начального пакета программ.

    1. Разберем основные пакеты которые нам понадобятся.

      • base — Обязательная к установке базовая группа пакетов. Размер пакетов ~2-3 kB, сюда входят такие основные утилиты как grep, pacman и systemd.
      • base-devel — Не обязательная, но крайне желательная группа пакетов для работы с AUR (Arch User Repository — поддерживаемое сообществом хранилище программ для пользователей Arch).
      • linux-zen — Zen ядро linux. Обязательный к установке, основной компонент системы. Можно заменить «ванильным» ядром, для этого вместо linux-zen пишем linux, или ядром с долгосрочной поддержкой, для этого вместо linux-zen пишем linux-lts.
      • linux-zen-headers — Cкрипты для ядра Linux. Обязательный к установке, основной компонент системы. Если решили использовать «ванильное» ядро, то меняем на linux-headers, если ядро с долгосрочной поддержкой, то меняем на linux-lts-headers.
      • linux-firmware — Не обязательный, но крайне желательный пакет с прошивками к устройствам.
      • nano — Не обязательный, но крайне желательный консольный блокнот.
      • dhcpcd — Не обязательный, но крайне желательный DHCP-клиент.
      • iwd — Wi-Fi клиент, если не планируется подключение к wifi, то не нужен.
      #: pacstrap -K /mnt base base-devel linux-zen linux-zen-headers linux-firmware nano dhcpcd iwd

    2. Скопируем/перенесем файл монтирования дисков. Исходя из записей в этом файле, диски и разделы при загрузке будут автоматически монтироваться к заданным директориям, можно редактировать в любой момент времени.

      #: genfstab -p -U /mnt >> /mnt/etc/fstab

    3. Перейдем в командную строку установленной системы.

      #: arch-chroot /mnt
